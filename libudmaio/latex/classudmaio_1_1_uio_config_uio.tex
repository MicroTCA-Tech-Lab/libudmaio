In a case where there are more than one memory mappings, an individual memory mapping can be selected by appending a colon (\char`\"{}\+:\char`\"{}) to the UIO name, followed by the mapping index. The UIO names are (in most cases) derived from the IP names in Vivado Block Diagram, and those names cannot include a colon. Additionally, a colon cannot be used in the device tree identifiers. It is, therefore, safe to reserve a colon as a special character.

Example\+:

Consider the following output of the {\ttfamily lsuio} command\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{uio8: name=example\_clocking, version=devicetree, events=0}
\DoxyCodeLine{    map[0]: addr=0xA0140000, size=65536}
\DoxyCodeLine{uio7: name=example\_app\_top, version=devicetree, events=0}
\DoxyCodeLine{    map[0]: addr=0xA0200000, size=1048576}
\DoxyCodeLine{    map[1]: addr=0xA0300000, size=65536}

\end{DoxyCode}



\begin{DoxyItemize}
\item The \char`\"{}example\+\_\+clocking\char`\"{} UIO can be selected either with {\ttfamily example\+\_\+clocking} or {\ttfamily example\+\_\+clocking\+:0}.
\item Similarly, the first memory mapping of \char`\"{}example\+\_\+app\+\_\+top\char`\"{} can be selected either with {\ttfamily example\+\_\+app\+\_\+top} or with {\ttfamily example\+\_\+app\+\_\+top\+:0}.
\item Second memory mapping of \char`\"{}example\+\_\+app\+\_\+top\char`\"{} can only be selected with {\ttfamily example\+\_\+app\+\_\+top\+:1}
